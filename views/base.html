I found it at: http://layouts.ironmyers.com/100_percent_Layouts/layout/?id=49 For better or worse, it uses the YUI CSS framework. What is a CSS framework, you ask? http://en.wikipedia.org/wiki/CSS_framework It looks silly to me, but the premise is that different browsers made different choices; such as with spacing or whether a "h1" header should be 10 or 20 pixels tall. To get around this, YUI has a "reset" that un-styles all the elements, and then it adds them all back in a way that will give you the same values in various browsers. Whether this is good or bad is beyond me. It is, however, not compatible with jquery ui. This despite the fact that in this discussion someone says that it is: http://www.filamentgroup.com/lab/styling_buttons_and_toolbars_with_the_jquery_ui_css_framework/ Example incompatibilities are seen with the way the margins look on alert messages, or the sudden appearance of outlining bullets on the tabs. I've worked around these but would be happy enough with a solution that didn't involve using YUI-CSS.
